# LP Builder — 3md構造によるLP生成スキル

## トリガー

このスキルは以下のキーワードで発火する:
- 「LP作成」「LP生成」「ランディングページ作成」
- 「LP改善」「LP修正」「LP最適化」
- 「landing page」「LP builder」

## 概要

3つの固定mdファイル + 1つの可変入力でLPを生成する:

| ファイル | 役割 | 変更頻度 |
|---------|------|---------|
| `design-system.md` | カラー・フォント・コンポーネント・禁止事項 | 低（学習で進化） |
| `lp-structure.md` | 11セクション構成・レイアウト・レスポンシブ | 低（型として固定） |
| `content-template.md` | 穴埋めテンプレート（可変入力） | 毎回（案件ごと） |

## ワークフロー

### Phase 1: 入力収集

ユーザーから以下を収集する（対話形式）:

1. **プロダクト名** と **一行説明**
2. **ターゲット業種/ペルソナ**
3. **主要な課題3つ** と **ソリューション**
4. **料金プラン**（あれば）
5. **既存ブランドカラー**（あれば design-system.md を上書き）

### Phase 2: デザイン決定（既存パイプライン連携）

業種が指定された場合、以下の順で既存スキルと連携する:

1. **`ui-ux-pro-max`** を参照 → 業種別の最適スタイル・カラー・フォントを決定
2. 決定内容で `design-system.md` のカラー/フォントセクションを一時的にオーバーライド
3. オーバーライドは生成時のみ適用（元ファイルは変更しない）

### Phase 3: LP生成

1. `design-system.md` を読み込み（デザインルール）
2. `lp-structure.md` を読み込み（構造テンプレート）
3. `content-template.md` をユーザー入力で穴埋め
4. 3つを統合して **単一HTMLファイル** を生成
   - Tailwind CSS CDN 使用
   - レスポンシブ対応（モバイルファースト）
   - セマンティックHTML
5. 生成先: プロジェクトディレクトリまたはユーザー指定パス

### Phase 4: 本番実装（オプション）

ユーザーが本番実装を希望した場合:

1. **`vibe-design`** でワイヤーフレーム検証
2. **`ps-design-ui`** で shadcn/ui + Tailwind コンポーネントとして実装
3. Next.js プロジェクトの場合は App Router ページとして出力

### Phase 5: 差分学習サイクル（LP完成後）

1. 生成されたLPとベースライン（design-system.md + lp-structure.md）を比較
2. ユーザーが加えた手動修正を検出
3. 修正が **汎用的**（他のLPにも適用可能）な場合のみ、design-system.md に書き戻しを提案
4. ユーザーの承認後に書き戻す → **スキルが育つ**

## 注意事項

- 生成HTMLは **単一ファイル完結**（外部依存は CDN のみ）
- 画像は placeholder を使用（`https://placehold.co/` または SVG）
- アニメーションは design-system.md の規則に従う（3秒超禁止）
- アクセシビリティ: コントラスト比4.5:1、フォーカス表示、alt属性必須
