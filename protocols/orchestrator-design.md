# Repository Agent Orchestrator - Design Document v0.1

全リポジトリの専属エージェントを統括し、ユーザーの意図に応じて適切なリポジトリ・エージェントにルーティングするオーケストレーターの設計書。

**ステータス**: 設計段階（3リポジトリ以上に専属エージェントが配置された時点で実装開始）

---

## 配置

`~/.claude/agents/repo-orchestrator.md`（グローバルエージェント）

---

## 機能

### 1. ディレクトリ管理
- `~/.claude/mailbox/directory.json` を参照し、全リポジトリの能力を把握
- 各リポジトリの `registry.json` からエージェントの capabilities を取得

### 2. インテントルーティング
- ユーザーの要求を分類し、適切なリポジトリ・エージェントに振り分け
- 複数リポジトリにまたがる場合は並行問い合わせ

### 3. クロスリポジトリ調整
- 複数リポジトリのデータを組み合わせるタスクの分割・統合
- コンテンツ企画の横断的な整合性チェック

### 4. メールボックス監視
- 全 mailbox の状態を定期確認（SessionStart フック連携）
- 未処理メッセージのアラート

---

## ルーティングロジック

```
ユーザー要求 → インテント分類
  ├─ 就活体験の記事を書きたい → re-grit-concierge
  ├─ EntryBoost のマーケティング → syukatsu-mypage-concierge
  ├─ 両方のデータが必要 → 両エージェントに並行問い合わせ
  └─ どのリポジトリか不明 → directory.json で capabilities 検索
```

### ルーティング優先順位
1. ユーザーが明示的にリポジトリ/エージェントを指定 → そのまま転送
2. キーワードマッチ（capabilities ベース） → 最適なコンシェルジュに転送
3. 該当なし → ユーザーに確認

---

## 既存オーケストレーションとの関係

| 仕組み | スコープ | 用途 |
|--------|---------|------|
| 蟻群仿生型（multi-agent SKILL.md） | 同一セッション内 | 並行タスク分割 |
| repo-orchestrator（本設計） | 複数リポジトリ間 | リポジトリ横断の連携 |

両者は補完的であり、排他ではない。
repo-orchestrator 内部で並行処理が必要な場合は蟻群仿生型を使用可能。

---

## 実装のタイミング

| 条件 | 現状 | 必要数 |
|------|------|--------|
| 専属エージェント配置済みリポジトリ数 | 1 (re-grit) | 3以上 |

現時点では re-grit のみのため、プロトコルベースの直接通信で十分。
syukatsu-mypage に concierge を配置し、さらに1リポジトリ追加された時点で実装を開始する。
