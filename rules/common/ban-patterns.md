# Ban Patterns（禁止パターン）

> ソース: @tsubotax（76禁止パターン設計）、gstack /plan-design-review（AI Slop検出）、調査報告書 2026-03-19

Claude がコード・UI・ドキュメントを生成する際に**絶対にやってはいけないこと**を定義する。
「何をすべきか」より「何をしてはいけないか」の方が品質に直結する。

---

## コード生成

### TypeScript / JavaScript
- `any` 型の使用禁止 — `unknown` + 型ガードを使う
- 未使用の import を残さない
- `console.log` / `console.error` をデバッグ目的で残さない（ロガーを使う）
- コメントアウトされたコードを残さない — 削除する（git に履歴がある）
- `// TODO` を理由なく残さない — チケット番号 or 期限を付ける
- `as` によるキャスト乱用禁止 — 型ガードか型定義の修正で対応
- `enum` よりも `const` オブジェクト + `as const` を優先
- `try-catch` で `error` を無視（空の catch ブロック）しない

### Python
- `type: ignore` の理由なし使用禁止
- bare `except:` 禁止 — 具体的な例外型を指定
- グローバル変数への代入禁止 — 関数引数で渡す
- `print()` デバッグを残さない — `logging` を使う

### 全言語共通
- ハードコードされたシークレット（API キー、パスワード、トークン）禁止
- マジックナンバー禁止 — 名前付き定数を使う
- 深いネスト（4レベル超）禁止 — 早期リターンで平坦化
- 800行超のファイル禁止 — 分割する
- 50行超の関数禁止 — 分割する
- 既存ファイルの無断上書き禁止 — 追記 or 新規作成
- `.env` ファイルをコミット対象に含めない

## UI 生成（AI Slop 検出リスト）

### 絶対禁止（AI臭いデザインの典型）
- 紫グラデーション背景の乱用
- 3カラム均等フィーチャーグリッド（「AIが作りました」感）
- 全要素センター寄せのレイアウト
- 過度なドロップシャドウ（`shadow-lg` 以上の多用）
- 意味のないアニメーション（hover で全要素が動く等）
- `#000000` 純黒テキスト — `#1a1a1a` 等のソフトブラックを使う
- Lorem ipsum / placeholder テキストを残す
- 汎用ストック写真アイコンの使用
- 「Hero → Features → Testimonials → CTA」の定番LP構成をそのまま使う
- 全セクションに無意味なアイコンを付ける
- border-radius を極端に大きくする（`rounded-3xl` 等の乱用）
- 背景に意味のないブロブ/波形 SVG を配置

### デザインシステム違反
- 既存プロジェクトのデザインシステム（DESIGN.md）と矛盾するスタイルの使用
- shadcn/ui コンポーネントのデフォルトスタイルを CSS で上書き（Tailwind の `className` で調整する）
- カラーパレット外の色を勝手に使う
- フォントファミリーを勝手に変更する

### レスポンシブ
- モバイル対応を省略しない
- `px` 固定幅レイアウトの使用禁止 — `max-w-*` + `w-full` を使う
- テキストサイズの `!important` 上書き禁止

---

## ドキュメント生成
- ユーザーが頼んでいない README / ドキュメントファイルを勝手に作成しない
- 絵文字を勝手に入れない（ユーザーが明示的に要求した場合のみ）
- 「いかがだったでしょうか」系の定型文を使わない
- diff やコード全文を見せない — ファイルパスと変更概要のみ報告

---

## ファイル操作
- `rm -rf /`, `rm -rf ~`, `rm -rf .` 禁止
- `git push --force` to main/master 禁止
- `git reset --hard` を確認なしに実行しない
- パイプ経由のスクリプト実行（`curl | bash`）禁止
- `sudo` の使用禁止（settings.json でもブロック済み）
- `/dev/null` へのリダイレクトでエラーを握りつぶさない

---

## 運用ルール
- このファイルは継続的に更新する（実際に問題が発生したら追記）
- プロジェクト固有の禁止パターンは各プロジェクトの `CLAUDE.md` に記載
- 言語固有の禁止パターンは `rules/{language}/ban-patterns.md` に分離可能
