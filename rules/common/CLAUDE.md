# rules/common/ — 全言語共通ルール

全プロジェクトに適用される共通ルール群。

## ファイル一覧

### 常時ロード（unconditional）

全セッションで自動ロードされる。非コーディングタスク（秘書業務・会社管理等）でも有効。

| ファイル | 内容 |
|---------|-----|
| `agents.md` | エージェント使用ガイド・並列実行パターン |
| `development-workflow.md` | 研究→計画→TDD→レビュー→コミットフロー |
| `git-workflow.md` | コミットメッセージ形式・PRプロセス |
| `hooks.md` | フック種類・TodoWrite実践 |
| `performance.md` | モデル選択・コンテキスト管理・拡張思考 |

### パススコープ（paths: frontmatter 付き）

コードファイルを読み書きするセッションでのみロードされる。コンテキストトークン節約のため `paths:` YAML frontmatter で制御。

| ファイル | トリガーパターン | 内容 |
|---------|----------------|-----|
| `ban-patterns.md` | `**/*.{ts,tsx,js,...,css,scss,html}` | 禁止パターン（コード/UI/ドキュメント/ファイル操作） |
| `coding-style.md` | `**/*.{ts,tsx,js,...,h}` | イミュータビリティ・ファイル整理・エラーハンドリング |
| `patterns.md` | `**/*.{ts,tsx,js,...,h}` | スケルトンプロジェクト・リポジトリパターン |
| `security.md` | `**/*.{ts,tsx,js,...,h}` | 必須セキュリティチェック・シークレット管理 |
| `testing.md` | `**/*.{test,spec}.*`, `**/tests/**` 等 | 80%テストカバレッジ・TDD・トラブルシューティング |
